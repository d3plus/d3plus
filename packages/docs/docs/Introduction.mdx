## Getting Started (React)

Install the React wrapper package:

```sh
npm install @d3plus/react
```

Pass a configuration object to the `config` prop of any visualization component:

```jsx
import {Treemap} from "@d3plus/react";

const config = {
  data: [
    {id: "alpha", value: 29},
    {id: "beta",  value: 10}
  ],
  groupBy: "id",
  size: "value"
};

<Treemap config={config} />
```

Set shared defaults across all visualizations using the `D3plusContext` Provider. Per-component `config` props override globals on a key-by-key basis via deep merge.

```jsx
import React from "react";
import ReactDOM from "react-dom/client";
import {D3plusContext} from "@d3plus/react";
import App from "src/App.jsx";

const globalConfig = {
  title: "Shared Title for All Visualizations"
};

ReactDOM.createRoot(document.getElementById("viz")).render(
  <React.StrictMode>
    <D3plusContext.Provider value={globalConfig}>
      <App />
    </D3plusContext.Provider>
  </React.StrictMode>
);
```

## Vanilla JavaScript (CDN)

Load the core bundle directly via a script tag:

```html
<script src="https://cdn.jsdelivr.net/npm/@d3plus/core"></script>
```

Then use the global `d3plus` object:

```html
<script>
  new d3plus.Treemap()
    .config({
      data: [
        {id: "alpha", value: 29},
        {id: "beta",  value: 10}
      ],
      groupBy: "id",
      size: "value"
    })
    .render();
</script>
```


## Packages

D3plus is a modular monorepo. Install only what you need:

| Package | Description |
|---------|-------------|
| [@d3plus/react](https://www.npmjs.com/package/@d3plus/react) | React component wrappers for all chart types |
| [@d3plus/core](https://www.npmjs.com/package/@d3plus/core) | Charts, components, and shapes (the main engine) |
| [@d3plus/color](https://www.npmjs.com/package/@d3plus/color) | Color scales, defaults, and utilities |
| [@d3plus/data](https://www.npmjs.com/package/@d3plus/data) | Data loading, filtering, and manipulation |
| [@d3plus/dom](https://www.npmjs.com/package/@d3plus/dom) | DOM and browser utility functions |
| [@d3plus/export](https://www.npmjs.com/package/@d3plus/export) | SVG and image export |
| [@d3plus/format](https://www.npmjs.com/package/@d3plus/format) | Number and date formatting |
| [@d3plus/locales](https://www.npmjs.com/package/@d3plus/locales) | Translation dictionaries for i18n |
| [@d3plus/math](https://www.npmjs.com/package/@d3plus/math) | Math and geometry utilities |
| [@d3plus/text](https://www.npmjs.com/package/@d3plus/text) | Text measurement, wrapping, and fitting |
| [@d3plus/types](https://www.npmjs.com/package/@d3plus/types) | Unified TypeScript type definitions |

For example, to use just the number formatting utilities:

```sh
npm install @d3plus/format
```

---

```js
import {formatAbbreviate} from "@d3plus/format";
```

To get all d3plus types in one import (useful for typing config objects or function parameters):

```sh
npm install @d3plus/types
```

---

```ts
import type {Treemap, D3plusComponentProps} from "@d3plus/types";
```


## Accessibility

D3plus is built with accessibility in mind. Every visualization automatically includes ARIA attributes, semantic roles, and contextual labels for screen readers. Additional features include:

- **`role="img"`** on all SVG containers, with optional `<title>` and `<desc>` elements
- **Contextual `aria-label`** attributes on every shape, tailored per chart type (e.g. rank and value for Treemap, x/y coordinates for Plot)
- **Inline data tables** for screen readers via the `ariaHidden` configuration
- **Color contrast utilities** that automatically select legible text colors against any background
- **Localization** of all accessible text and number formatting across multiple locales

See the **Accessibility** page in the sidebar for full details, including a WCAG 2.1 AA conformance summary.
